# 🤖 Распределенный Telegram-бот расписания (Docker)

Бот для автоматического парсинга и отображения расписания с поддержкой распределенной архитектуры и работы через прокси. Оптимизирован для быстрого ответа и автоматического масштабирования.

## 🌟 Особенности
*   **Распределенная архитектура**: Отдельные сервисы для логики бота и парсинга данных.
*   **Быстрая загрузка**: Оптимизированные алгоритмы ожидания (без фиксированных пауз) и ускоренный опрос кэша (Redis).
*   **Авто-масштабирование**: По умолчанию запускается 3 независимых воркера для параллельной обработки запросов.
*   **Проксирование**: Весь внешний трафик туннелируется через единый прокси-узел.

## 🏗 Архитектура проекта
*   **bot**: Основной интерфейс Telegram и бизнес-логика.
*   **worker**: Скраперы на базе Playwright (парсинг сайта). Масштабируются автоматически.
*   **redis**: Хранилище кэша и очередь заданий.
*   **proxy**: SOCKS5-туннель для всех сетевых запросов.

## 📋 Требования
*   ОС с поддержкой Docker (Ubuntu, Debian, Windows и др.)
*   Docker и Docker Compose

## 🛠 Подготовка сервера (Ubuntu/Debian)

Перед установкой проекта убедитесь, что ваш сервер обновлен и на нем установлены все необходимые инструменты.

### 1. Обновление пакетов
```bash
sudo apt update && sudo apt upgrade -y
```

### 2. Установка утилит (Git, Nano)
```bash
sudo apt install -y git nano curl
```

### 3. Установка Docker и Docker Compose
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo apt-get install -y docker-compose-plugin
```

## 🚀 Установка и настройка

### 1. Клонирование репозитория
```bash
git clone https://github.com/Monopoly450/TG_Test.git
cd TG_Test
```

### 2. Настройка переменных окружения (`.env`)
Создайте и откройте файл `.env` в корневой директории проекта для редактирования:
```bash
nano .env
```

Внутри редактора заполните следующие параметры:
*   `BOT_TOKEN`: Токен вашего бота от [@BotFather](https://t.me/BotFather).
*   `LOGIN`: Логин (email) для доступа к системе расписания.
*   `PASSWORD`: Пароль для доступа к системе расписания.

**Как сохранить и выйти из редактора Nano:**
1. Нажмите `Ctrl + O` (английская О), затем `Enter`, чтобы сохранить изменения.
2. Нажмите `Ctrl + X`, чтобы закрыть редактор.

### 3. Настройка прокси (Опционально)
Если вам нужно изменить внешний прокси-сервер, отредактируйте `docker compose.yml` в секции `proxy` (команда запуска `gost`).

### 4. Запуск проекта
Соберите и запустите все сервисы одной командой:
```bash
docker compose up -d --build
```

### 5. Мониторинг и управление
Проверить статус контейнеров:
```bash
docker compose ps
```
Просмотреть логи (например, для бота):
```bash
docker compose logs -f bot
```

## ⚙️ Управление воркерами
Проект настроен на автоматический запуск **3 воркеров** (в `docker compose.yml` указано `replicas: 3`). Это позволяет обрабатывать несколько запросов расписания одновременно без задержек.

Если вам нужно изменить количество воркеров "на лету":
```bash
docker compose up -d --scale worker=5
```

## 🛑 Остановка и выключение

Для безопасного выключения сервера или временной остановки бота используйте следующие команды:

### Остановка всех контейнеров (Рекомендуется):
```bash
docker compose stop
```
*Эта команда просто останавливает процессы. Сами контейнеры остаются в системе, их не нужно пересоздавать при следующем запуске. Это самый быстрый и безопасный способ перед выключением сервера.*

### Запуск после остановки:
```bash
docker compose start
```

### Полная остановка и удаление (Если нужно обновить проект):
```bash
docker compose down
```
*Используйте эту команду, если вы внесли изменения в `docker compose.yml` или хотите полностью очистить ресурсы. Данные в Volume (базы данных и кэш) при этом **сохраняются**.*

### Остановка конкретного сервиса:
```bash
docker compose stop bot
```

## 📂 Структура данных
*   `/data`: Данные пользователей и конфигурация техработ (Docker Volume `bot_data`).
*   `/cache`: Кэшированные данные расписаний (Docker Volume `shared_cache`).
*   `/redis`: База данных Redis (Docker Volume `redis_data`).

Все данные сохраняются при перезапуске или обновлении контейнеров благодаря томам Docker.
