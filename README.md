# Распределенный Telegram-бот расписания (Docker)

Бот для парсинга и отображения расписания с поддержкой распределенной архитектуры и работы через прокси.

## Архитектура
Проект разделен на независимые контейнеры:
*   **bot**: Интерфейс Telegram и логика.
*   **worker**: Скраперы на базе Playwright (парсинг сайта). Можно масштабировать количество воркеров.
*   **redis**: Очередь задач и кэш данных.
*   **proxy**: Единый выходной узел через SOCKS5 (туннелирование всего трафика).

## Требования
*   Ubuntu (или любая ОС с Docker)
*   Docker и Docker Compose

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/Monopoly450/TG_Test.git
cd TG_Test
```

### 2. Настройка переменных окружения
Откройте `docker-compose.yml` и отредактируйте следующие параметры:

**В секции `proxy`:**
*   `GLOBAL_PROXY`: ваш внешний SOCKS5 прокси в формате `socks5://user:pass@host:port`

**В секциях `bot` и `worker` (по желанию):**
*   `BOT_TOKEN`: токен вашего бота от @BotFather
*   `LOGIN` и `PASSWORD`: учетные данные для доступа к сайту расписания

### 3. Запуск проекта
Соберите и запустите все контейнеры одной командой:
```bash
docker-compose up -d --build
```

### 4. Управление воркерами (Масштабирование)
Если нужно ускорить загрузку расписаний при большой нагрузке, увеличьте количество воркеров:
```bash
docker-compose up -d --scale worker=3
```

## Структура данных
*   `/data`: пользовательские данные и настройки техработ (сохраняются в Docker Volume `bot_data`).
*   `/cache`: кэшированные расписания (сохраняются в Docker Volume `shared_cache`).
*   `/redis`: данные базы Redis (сохраняются в Docker Volume `redis_data`).

Все данные сохраняются при перезапуске или обновлении контейнеров.
